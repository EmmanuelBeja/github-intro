Github: Getting started.

Prerequisites

  • Internet.
  • A github Account.
  • For windows;gitbash. Download gitbash here

Creating A Repository

  • Create repository in your github Account and call it the name of the project.
  • Add description of project.(optional)
  • Check public(For public repository. Private is paid for.)
  • Initialize with a readme.(Optional)
  • Add .gitignore file (optional)
  • Add license(commonly used license for open source projects is MIT. This is also optional)
  • Copy link of the project. It could be in either of the two options bellow.

OR


Cowabunga! You have your first github repository. Simple right? So now we need to put stuff in it.

Pushing to Github

Method 1

On your computer open the terminal for linux users. For window users, open gitbash comandline tool.
Ensure it is in the place you want to work your project on. Eg. Desktop
now key in the following commands:

$ git clone clonelinkfromgithub


This will download the repository you created on github.
Now get into the downloaded project folder.I will do it as bellow

$ cd nameofproject


Copy files you want to upload in the folder you have cloned. Ensuring your terminal/gitbash console is in the cloned project folder follow the steps bellow. You will do this for each and every file. One at a time. Given an example; having files index.py and login.py, we will start with either of the files and follow the procedure bellow. When done, we will redo the same procces with the second remaining file. This is something we will do for each file again and again untill we have no more remaining.

$ git add index.py
$ git commit -m "describe what you did"
$ git push -u origin master


Wait...What just happened?
git add index.py
Adds the modified and new (untracked) file in the current directory to the staging area (a.k.a. the index), thus preparing it to be included in the next git commit.
git commit -m "describe what you did"
Record changes to the repository. Note: never let the mssg have a ' in between
git push -u origin master
Push changes from the local master branch to the remote master branch. This might prompt you for you your github user name and password depending on configuration of github on your machine.
Use "git status" incase you are lost and need hints from github
